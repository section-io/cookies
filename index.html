Welcome

HTTP cookies exist to communicate state between a user-agent (eg web browser) and a web server, typically for maintaining information about authentication or eg what items a user has in their shopping cart. Traditionally web servers return user-specific content in response to the user's cookies and so caching proxies, like Varnish Cache, will by default avoid caching the responses to requests containing cookies. This is a safe default to avoid serving to a user content that was original intended for another user.

However, HTTP cookies are also very heavily used to track state that is never consumed by the origin web server and will never affect the content returned. Third-party marketing cookies are a popular example. To improve the cacheability of responses from a web server, a caching proxy needs to know the difference between cookies that can be safely ignored and those that should cause a response to be uncacheable.

This site aims to document the popular cookies found on many sites and provide configuration file fragments to be used with popular caching proxy implementations to improve HTTP response cacheability.


# VCL for Varnish 3.x without libvmod_cookie.
# Add to sub vcl_recv
if (req.http.Cookie) {
	set req.http.Cookie = regsuball(req.http.Cookie, "(^|; ) *RT=[^;]+;? *", "\1");

	# GoSquared
	set req.http.Cookie = regsuball(req.http.Cookie, "(^|; ) *gs_._GSN-[^;]*=[^;]+;? *", "\1");

	# Google Analytics
	set req.http.Cookie = regsuball(req.http.Cookie, "(^|; ) *_ga=[^;]+;? *", "\1");
	set req.http.Cookie = regsuball(req.http.Cookie, "(^|; ) *__utm..?=[^;]+;? *", "\1");

	# Yahoo Search
	set req.http.Cookie = regsuball(req.http.Cookie, "(^|; ) *ysm_[^=]+=[^;]+;? *", "\1");

	if (req.http.Cookie == "") {
		remove req.http.Cookie;
	}
}
